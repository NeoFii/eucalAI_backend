    2. class RelayBillingService with all @staticmethod methods:

       - async def get_balance(cache_redis: aioredis.Redis, user_id: int, db_session_factory) -> int:
         * Try Redis GET user:quota:{user_id}. If exists, return int(value).
         * If Redis miss or error (D-06): use db_session_factory to query User.balance from DB, then SET user:quota:{user_id} with ex=300 (lazy init per Open Question 2).
         * Return balance in micro-units.

       - async def estimate_cost(model_prices: dict, model: str, max_tokens: int | None) -> int:
         * Per D-05: output_price_per_million * min(max_tokens or 4096, 4096) / 1_000_000 + input_price_per_million * 2048 / 1_000_000.
         * If model not in model_prices, return settings.RELAY_BILLING_FALLBACK_COST.
         * Return int (micro-units).

       - async def pre_consume(cache_redis: aioredis.Redis, user_id: int, estimated_cost: int, balance: int, trust_quota: int) -> tuple[int, bool]:
         * Per D-04: if balance > trust_quota, return (0, True) — trusted, no pre-consume.
         * Otherwise: DECRBY user:quota:{user_id} by estimated_cost.
         * If new_balance < 0: INCRBY to rollback, raise InsufficientBalanceError.
         * Return (estimated_cost, False).
         * On Redis error (D-06): log warning, return (0, True) — degrade to trusted mode.

       - async def settle(cache_redis: aioredis.Redis, user_id: int, pre_consumed: int, actual_cost: int, trusted: bool) -> None:
         * Per D-01 settle logic:
           - If trusted: DECRBY user:quota:{user_id} by actual_cost.
           - Else: compute delta = actual_cost - pre_consumed. If delta > 0: DECRBY delta. If delta < 0: INCRBY abs(delta). If 0: no-op.
         * On Redis error: log warning (DB persist will handle via create_task).

       - async def refund(cache_redis: aioredis.Redis, user_id: int, pre_consumed: int) -> None:
         * If pre_consumed > 0: INCRBY user:quota:{user_id} by pre_consumed.
         * On Redis error: log warning.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> require_api_key | Untrusted Bearer token crosses into auth validation |
| Redis cache -> application | Cached token data could be stale (60s TTL mitigates) |
| Application -> DB | Authoritative source, trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | relay/auth.py | mitigate | SHA256 hash comparison + TTLCache limits lookup frequency; disabled keys actively invalidated via invalidate_api_key_cache |
| T-06-02 | Tampering | relay/billing.py | mitigate | Redis DECRBY atomic operation prevents read-modify-write race; negative balance check with rollback |
| T-06-03 | Information Disclosure | relay/auth.py | mitigate | ValidatedApiKey does not store raw key; only hash used for cache keys |
| T-06-04 | Elevation of Privilege | relay/auth.py | mitigate | Disabled key active DEL token:{hash} + 60s TTL ensures max 60s stale access (D-07) |
| T-06-05 | Denial of Service | relay/billing.py | accept | Redis DECRBY is O(1); TTLCache bounds memory; no amplification vector |
</threat_model>
