<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application -> inference-service | HTTP call to external GPU server with shared secret |
| asyncio.create_task -> DB | Background task writes to DB independently of request lifecycle |
| ChannelSelector state -> request | In-memory state affects routing decisions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Information Disclosure | relay/inference_client.py | mitigate | X-Inference-Secret header never logged; only passed in HTTP header to inference-service |
| T-06-11 | Tampering | relay/call_log_writer.py | accept | Fire-and-forget means log write can fail silently; billing settle retries 3x to ensure correctness |
| T-06-12 | Denial of Service | relay/channel_selector.py | mitigate | Auto-disable after 5 failures prevents repeated calls to broken channels; cooldown prevents thundering herd |
| T-06-13 | Spoofing | relay/routing.py | mitigate | Only models in user_facing_aliases accepted; unknown models rejected with 400 |
| T-06-14 | Tampering | relay/channel_affinity.py | accept | Affinity is a performance optimization; worst case is suboptimal channel selection, not security breach |
</threat_model>
