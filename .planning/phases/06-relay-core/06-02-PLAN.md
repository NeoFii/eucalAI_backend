           * Per D-09: GET routing_config:version from Redis.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis routing_config:version -> application | Version number from Redis triggers DB reload |
| DB routing_settings -> normalize_runtime_config | Raw DB rows transformed into runtime config |
| Upstream URL validation | Channel api_base URLs validated against SSRF blocklist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering | relay/config_cache.py | mitigate | routing_settings only writable by admin (ADMIN-06); version key only INCR'd by admin service layer |
| T-06-07 | Tampering | relay/upstream.py | mitigate | _validate_upstream_url blocks localhost, private networks, metadata endpoints (SSRF prevention) |
| T-06-08 | Information Disclosure | relay/upstream.py | mitigate | api_key from pool_accounts stored encrypted; decrypted only at resolve time, never logged |
| T-06-09 | Denial of Service | relay/config_cache.py | accept | Version poll is single Redis GET per request (<1ms); DB reload only on version mismatch (rare) |
</threat_model>
