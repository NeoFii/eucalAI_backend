# Plan: Lifespan skeleton and health endpoint

## Objective

实现 LifespanRegistry 资源管理器、FastAPI 应用入口（main.py）、路由注册骨架、/health 和 /ready 端点。完成后 `uvicorn api_service.main:app` 可启动并响应健康检查请求。

## Tasks

<task id="1">
<title>实现 LifespanRegistry</title>
<read_first>
- .planning/phases/01-project-scaffold-common-layer/01-RESEARCH.md (Section 5 — LifespanRegistry Pattern)
- .planning/phases/01-project-scaffold-common-layer/01-CONTEXT.md (D-15, D-16)
</read_first>
<action>
在 services/api-service/api_service/core/lifespan.py 创建 LifespanRegistry 类：

1. 定义 `@dataclass _Resource`：name: str, init_fn: Callable[[], Awaitable[None]], shutdown_fn: Callable[[], Awaitable[None]] | None, priority: int

2. 定义 `@dataclass LifespanRegistry`：
   - `_resources: list[_Resource]` (default_factory=list)
   - `_initialized: list[str]` (default_factory=list)
   - `register(name, init_fn, shutdown_fn=None, priority=100)` 方法
   - `async startup()` 方法：按 priority 升序执行 init_fn，失败时 fail-fast 并调用 _cleanup
   - `async shutdown()` 方法：调用 _cleanup
   - `async _cleanup()` 方法：按 priority 降序执行已初始化资源的 shutdown_fn

3. 使用 `logging.getLogger(__name__)` 记录日志
4. startup 中每个资源初始化成功后 logger.info("resource_initialized", extra={"resource": name})
5. startup 中资源失败时 logger.critical("resource_init_failed", extra={"resource": name}, exc_info=True)
6. _cleanup 中 shutdown 失败时 logger.warning("resource_shutdown_failed", extra={"resource": name}, exc_info=True)
</action>
<acceptance_criteria>
- `api_service/core/lifespan.py` 包含 `class LifespanRegistry` 定义
- LifespanRegistry 有 `register`, `startup`, `shutdown` 公开方法
- `register` 接受 name: str, init_fn, shutdown_fn (可选), priority: int (默认 100)
- `startup` 按 priority 升序执行
- `shutdown` / `_cleanup` 按 priority 降序执行
- 初始化失败时先清理已初始化资源再 raise（fail-fast 策略）
- 使用 `from __future__ import annotations` 和 `from dataclasses import dataclass, field`
</acceptance_criteria>
</task>

<action>
在 services/api-service/api_service/main.py 创建 FastAPI 应用入口：

1. 顶部导入并立即调用 configure_logging_from_settings(settings)（在 app 创建前）

2. 创建 LifespanRegistry 实例 `registry`

3. 定义 Phase 1 初始化函数：
   - `async _init_logging()`: 调用 configure_logging_from_settings(settings)（实际上已在模块级调用，此处为 registry 形式化注册，可以是 no-op 或重新调用）
   - `async _init_snowflake()`: 调用 configure_snowflake(worker_id=settings.SNOWFLAKE_WORKER_ID, datacenter_id=settings.SNOWFLAKE_DATACENTER_ID)

4. 注册资源：
   - registry.register("logging", init_fn=_init_logging, priority=0)
   - registry.register("snowflake", init_fn=_init_snowflake, priority=10)

5. 定义 lifespan context manager：
   ```python
   @asynccontextmanager
   async def lifespan(app: FastAPI):
       await registry.startup()
       log_event(logger, logging.INFO, "service_started", service=settings.SERVICE_NAME)
       yield
       await registry.shutdown()
       log_event(logger, logging.INFO, "service_stopped", service=settings.SERVICE_NAME)
   ```

6. 创建 FastAPI app：
   - title=settings.PROJECT_NAME
   - description=settings.DESCRIPTION
   - version=settings.VERSION
   - docs_url="/docs" if settings.DEBUG else None
   - redoc_url="/redoc" if settings.DEBUG else None
   - openapi_url="/openapi.json" if settings.DEBUG else None
   - lifespan=lifespan
   - redirect_slashes=False

7. 添加中间件（按 RESEARCH.md Section 8.4 顺序）：
   - app.add_middleware(CORSMiddleware, allow_origins=settings.cors_allowed_hosts, allow_credentials=True, allow_methods=["*"], allow_headers=["*"])
   - install_observability(app, service_name=settings.SERVICE_NAME)

8. 注册异常处理器：register_exception_handlers(app)

9. 挂载路由：app.include_router(api_router)

10. 定义健康检查端点：
    - GET /health: 返回 {"status": "healthy", "service": settings.SERVICE_NAME, "version": settings.VERSION}
    - GET /ready: Phase 1 返回 {"status": "ready", "service": settings.SERVICE_NAME}（Phase 2 将改为调用 build_readiness_response）
</action>
