# Plan 02-03: Snowflake ID with per-worker safety

<objective>
确保 Snowflake ID 生成器在 uvicorn 多 worker（fork）场景下为每个进程分配不同的 worker_id，消除 ID 碰撞风险。通过 `os.getpid() % 32` 在 lifespan startup 时动态计算 worker_id，并编写测试验证多进程场景下 ID 唯一性。
</objective>

<must_haves>
## Goal-Backward Verification

**truths** (must be true after execution):
- `_init_snowflake` 使用 `os.getpid() % 32` 计算 worker_id（不再使用静态 `settings.SNOWFLAKE_WORKER_ID`）
- `datacenter_id` 保持使用 `settings.SNOWFLAKE_DATACENTER_ID`（默认 1）
- 测试文件验证：4 个不同 PID 产生 4 个不同的 instance_id
- 测试文件验证：使用不同 worker_id 的生成器产生的 ID 无碰撞

**anti-goals** (must NOT happen):
- 不得依赖 Redis 分配 worker_id（Snowflake priority=10 在 Redis priority=30 之前）
- 不得修改 `api_service/common/utils/snowflake.py` 的公共 API
- 不得移除 `settings.SNOWFLAKE_WORKER_ID` 配置项（保留用于单 worker 调试场景）
</must_haves>

<tasks>

### Task 1: 修改 _init_snowflake 使用动态 worker_id

<read_first>
- services/api-service/api_service/main.py
- services/api-service/api_service/common/utils/snowflake.py
</read_first>

<action>
修改 `services/api-service/api_service/main.py` 中的 `_init_snowflake` 函数：

1. 在文件顶部添加 `import os`（如果尚未存在）
2. 将函数体改为：
   ```
   worker_id = os.getpid() % 32
   configure_snowflake(worker_id=worker_id, datacenter_id=settings.SNOWFLAKE_DATACENTER_ID)
   ```
3. 添加 logger.info 记录分配的 worker_id：
   ```
   logger.info("snowflake_configured", extra={"worker_id": worker_id, "pid": os.getpid()})
   ```

注意：此修改可能已在 Plan 02-01 Task 4 中完成。如果已完成，验证实现正确即可。
</action>

<acceptance_criteria>
- `_init_snowflake` 函数体包含 `os.getpid() % 32`
- 不再引用 `settings.SNOWFLAKE_WORKER_ID`（在 `_init_snowflake` 内部）
- `datacenter_id` 参数使用 `settings.SNOWFLAKE_DATACENTER_ID`
</acceptance_criteria>

### Task 2: 编写多 worker Snowflake ID 唯一性测试

<read_first>
- services/api-service/api_service/common/utils/snowflake.py
</read_first>

<action>
创建 `services/api-service/tests/test_snowflake_worker.py`：

1. 测试 `test_different_pids_produce_different_worker_ids`：
   - 模拟 4 个不同的 PID（如 1000, 1001, 1002, 1003）
   - 对每个 PID 计算 `pid % 32`
   - 断言 4 个结果互不相同

2. 测试 `test_different_workers_produce_unique_ids`：
   - 使用 worker_id=0,1,2,3 分别创建 `SnowflakeIDGenerator` 实例
   - 每个实例生成 100 个 ID
   - 断言所有 400 个 ID 互不重复（使用 set 长度验证）

3. 测试 `test_pid_mod_32_range`：
   - 验证 `os.getpid() % 32` 结果在 0-31 范围内
   - 验证 `_get_instance_id()` 在 configure_snowflake 后返回 `datacenter_id * 32 + worker_id`

4. 测试 `test_configure_snowflake_clears_cache`：
   - 调用 `configure_snowflake(worker_id=5, datacenter_id=1)`
   - 获取 generator，验证 instance_id 为 `1 * 32 + 5 = 37`
   - 调用 `configure_snowflake(worker_id=10, datacenter_id=1)`
   - 获取新 generator，验证 instance_id 为 `1 * 32 + 10 = 42`

使用 pytest 框架，无需 pytest-asyncio（纯同步测试）。
</action>

<acceptance_criteria>

- 测试覆盖：不同 PID 产生不同 worker_id、不同 worker 产生唯一 ID、cache_clear 生效
</acceptance_criteria>
