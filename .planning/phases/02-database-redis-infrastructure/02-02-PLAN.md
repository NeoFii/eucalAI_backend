# Plan 02-02: Redis connection pools (3 logical DBs)

<objective>
在 api-service lifespan 中注册 Redis db/0（session/rate-limit）和 db/2（cache）的初始化与关闭，将 /ready 端点扩展为同时检查 DB 和 Redis 健康状态。db/1（ARQ worker queue）由 ARQ worker 进程独立管理，api-service 不需要初始化。
</objective>

<must_haves>
## Goal-Backward Verification

**truths** (must be true after execution):
- lifespan registry 包含 "redis" 资源（priority=30），init_fn 调用 `init_redis(settings.REDIS_URL)`，shutdown_fn 调用 `close_redis()`
- lifespan registry 包含 "cache_redis" 资源（priority=30），init_fn 调用 `init_cache_redis(settings.CACHE_REDIS_URL)`，shutdown_fn 调用 `close_cache_redis()`
- `/ready` 端点同时检查 DB + Redis(db/0) + Cache Redis(db/2)，任一失败返回 503
- Redis 启动时执行 ping 验证连接（已内置于 `init_redis` / `init_cache_redis`）

**anti-goals** (must NOT happen):
- 不得为 db/1（ARQ）创建连接池或健康检查（ARQ worker 是独立进程）
- 不得修改 `api_service/common/infra/redis.py` 或 `cache.py` 的实现
- 不得设置 `max_connections` 参数（保持 redis-py 默认行为）
</must_haves>

<tasks>

### Task 1: 注册 Redis db/0 lifespan 资源

<read_first>
- services/api-service/api_service/main.py
- services/api-service/api_service/common/infra/redis.py
</read_first>

<action>
修改 `services/api-service/api_service/main.py`，添加：

1. `_init_redis` 异步函数：
   - 从 `api_service.common.infra.redis` 导入 `init_redis`
   - 调用 `await init_redis(settings.REDIS_URL)`

2. `_shutdown_redis` 异步函数：
   - 从 `api_service.common.infra.redis` 导入 `close_redis`
   - 调用 `await close_redis()`

3. 注册：
   `registry.register("redis", init_fn=_init_redis, shutdown_fn=_shutdown_redis, priority=30)`
</action>

<acceptance_criteria>
- `registry._resources` 包含 name="redis" 的资源
- priority=30，shutdown_fn 不为 None
- `_init_redis` 内部调用 `init_redis(settings.REDIS_URL)`
</acceptance_criteria>

### Task 3: 升级 /ready 端点添加 Redis 健康检查

<read_first>
- services/api-service/api_service/main.py
- services/api-service/api_service/common/health.py
- services/api-service/api_service/common/infra/redis.py
- services/api-service/api_service/common/infra/cache.py
</read_first>

<action>
修改 `/ready` 端点（在 Plan 02-01 已升级为 DB 检查的基础上），扩展 redis_check 参数：

1. 定义 `_combined_redis_check` 异步函数：
   - 从 `api_service.common.infra.redis` 导入 `check_redis_ready`
   - 从 `api_service.common.infra.cache` 导入 `check_cache_redis_ready`
   - 先调用 `check_redis_ready()`，如果失败返回 `(False, f"Redis db/0: {err}")`
   - 再调用 `check_cache_redis_ready()`，如果失败返回 `(False, f"Cache Redis db/2: {err}")`
   - 两者都成功返回 `(True, None)`

2. 在 `build_readiness_response` 调用中传入 `redis_check=_combined_redis_check`

最终 `/ready` 端点签名：
```
return await build_readiness_response(
    service_name=settings.SERVICE_NAME,
    database_check=_db_check,
    redis_check=_combined_redis_check,
)
```
</action>

<acceptance_criteria>
- `/ready` 返回 JSON 包含 `"checks": {"database": {...}, "redis": {...}}`
- Redis db/0 不可用时，`checks.redis.status` 为 `"error"`，HTTP 状态码 503
- Cache Redis db/2 不可用时，`checks.redis.status` 为 `"error"`，HTTP 状态码 503
- 两者都可用时，`checks.redis.status` 为 `"ok"`，HTTP 状态码 200
</acceptance_criteria>
