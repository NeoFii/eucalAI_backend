# Plan 02-01: SQLAlchemy async engine and session factory

<objective>
创建 api-service 的数据库运行时模块，实例化 ServiceDatabaseRuntime 并导出便捷函数（get_db, close_db 等），在 lifespan 中注册 DB engine 初始化和关闭，覆盖连接池参数为 2h4g 服务器安全值（pool_size=5, max_overflow=10），并将 /ready 端点升级为真实的 DB 健康检查。
</objective>

<must_haves>
## Goal-Backward Verification

**truths** (must be true after execution):
- `api_service.core.db` 模块导出 `create_engine`, `get_engine`, `init_session_factory`, `get_db`, `get_db_context`, `close_db`, `Base`
- `ApiServiceSettings.DATABASE_POOL_SIZE` 默认值为 5
- `ApiServiceSettings.DATABASE_MAX_OVERFLOW` 默认值为 10
- lifespan registry 包含 "database" 资源，priority=20，有 init_fn 和 shutdown_fn
- `/ready` 端点调用 `check_database_ready` 验证 DB 连接，失败时返回 503
- `api_service/db.py` 代理模块存在，导出 `Base`（供 Alembic `_env_shared.py` 使用）

**anti-goals** (must NOT happen):
- 不得重新定义 Base（必须使用 `api_service.common.infra.db.base.Base`）
- 不得修改 `ServiceDatabaseRuntime` 类本身
- 不得硬编码 DATABASE_URL（必须从 settings 读取）
- pool_size 不得超过 5，max_overflow 不得超过 10（2h4g 服务器约束）
</must_haves>

### Task 1: 创建 api_service/core/db.py

<read_first>
- services/api-service/api_service/common/infra/db/runtime.py
- services/api-service/api_service/common/infra/db/base.py
</read_first>

<action>
创建 `services/api-service/api_service/core/db.py`：

1. 从 `api_service.common.infra.db.base` 导入 `Base`
2. 从 `api_service.common.infra.db.runtime` 导入 `ServiceDatabaseRuntime`
3. 实例化 `_runtime = ServiceDatabaseRuntime(Base)`
4. 导出以下便捷函数（赋值为 `_runtime.xxx`）：
   - `create_engine`
   - `get_engine`
   - `init_session_factory`
   - `get_db`
   - `get_db_context`
   - `close_db`
5. 同时导出 `Base`
6. 定义 `__all__` 列表包含所有导出名称
7. 模块 docstring: `"""Database runtime for api-service — single engine, single Base."""`
</action>

<acceptance_criteria>
- 文件 `services/api-service/api_service/core/db.py` 存在
- `from api_service.core.db import get_db, close_db, create_engine, get_engine, init_session_factory, get_db_context, Base` 不报错
- `Base` 是 `api_service.common.infra.db.base.Base` 的同一对象（`is` 关系）
- `_runtime` 是 `ServiceDatabaseRuntime` 实例
</acceptance_criteria>

### Task 3: 覆盖 ApiServiceSettings 连接池默认值

<read_first>
- services/api-service/api_service/core/config.py
- services/api-service/api_service/common/config.py
</read_first>

<action>
修改 `services/api-service/api_service/core/config.py`，在 `ApiServiceSettings` 类的 `# ── Database` 区域添加：

```
DATABASE_POOL_SIZE: int = 5
DATABASE_MAX_OVERFLOW: int = 10
```

这两行放在 `DATABASE_URL` 定义之后。这些值覆盖 `BaseServiceSettings` 的默认值 10/20，确保 4 workers × 15 = 60 连接在 MySQL 151 限制内。
</action>

<acceptance_criteria>
- `settings.DATABASE_POOL_SIZE` 返回 5
- `settings.DATABASE_MAX_OVERFLOW` 返回 10
- 未设置环境变量时使用这些默认值
</acceptance_criteria>

### Task 4: 注册 DB lifespan 资源并升级 /ready 端点

<read_first>
- services/api-service/api_service/main.py
- services/api-service/api_service/common/health.py
- services/api-service/api_service/core/lifespan.py
</read_first>

<action>
修改 `services/api-service/api_service/main.py`：

1. 添加 `_init_database` 异步函数：
   - 从 `api_service.core.db` 导入 `create_engine`, `init_session_factory`
   - 调用 `create_engine(settings.DATABASE_URL, echo=settings.DATABASE_ECHO, pool_size=settings.DATABASE_POOL_SIZE, max_overflow=settings.DATABASE_MAX_OVERFLOW, pool_recycle=settings.DATABASE_POOL_RECYCLE, pool_timeout=settings.DATABASE_POOL_TIMEOUT)`
   - 调用 `init_session_factory()`

2. 添加 `_shutdown_database` 异步函数：
   - 从 `api_service.core.db` 导入 `close_db`
   - 调用 `await close_db()`

3. 注册到 registry：
   `registry.register("database", init_fn=_init_database, shutdown_fn=_shutdown_database, priority=20)`

4. 替换 `/ready` 端点实现：
   - 从 `api_service.common.health` 导入 `build_readiness_response`, `check_database_ready`
   - 从 `api_service.core.db` 导入 `get_engine`
   - 定义内部 `_db_check` 异步函数调用 `check_database_ready(get_engine)`
   - 调用 `build_readiness_response(service_name=settings.SERVICE_NAME, database_check=_db_check)`
   - 注意：redis_check 参数暂不传（Plan 02-02 会补充）

5. 修改 `_init_snowflake` 函数：
   - 添加 `import os`
   - 将 `worker_id=settings.SNOWFLAKE_WORKER_ID` 改为 `worker_id=os.getpid() % 32`
   - 保持 `datacenter_id=settings.SNOWFLAKE_DATACENTER_ID`
</action>

<acceptance_criteria>
- `registry._resources` 包含 name="database" 的资源，priority=20
- `_init_database` 使用 `settings.DATABASE_POOL_SIZE`（值为 5）和 `settings.DATABASE_MAX_OVERFLOW`（值为 10）
- `_shutdown_database` 调用 `close_db()`
- `/ready` 端点返回 JSON 包含 `"checks": {"database": {"status": ...}}`
- DB 不可用时 `/ready` 返回 HTTP 503
- `_init_snowflake` 使用 `os.getpid() % 32` 作为 worker_id
</acceptance_criteria>

</tasks>
