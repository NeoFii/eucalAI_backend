逻辑（从 user-service dependencies.py 迁移）：
1. 从 Authorization Header (Bearer) 或 Cookie (`user_access_token`) 提取 token
2. 调用 `decode_token(token, settings.JWT_SECRET_KEY, settings.JWT_ALGORITHM)`

5. 通过 `UserRepository(db).get_by_uid(uid)` 查询用户
6. 用户不存在时抛出 `UserNotFoundException`
7. 调用 `set_uid(user.uid)` 设置 observability context
8. 返回 `User` 实例

关键差异（D-08）：不做 token blacklist 检查（user-service 原实现也不做）。

<action>
创建 `api_service/core/dependencies/admin.py`，实现三个依赖函数。

### get_current_admin

逻辑（从 admin-service dependencies.py 迁移）：
1. 从 Authorization Header (Bearer) 或 Cookie (`admin_access_token`) 提取 token
2. 调用 `is_token_blacklisted(get_token_jti(token))`（D-07：保留 blacklist 检查）
3. 如果 blacklisted，抛出 `InvalidTokenException(detail="令牌已被吊销")`
4. 调用 `decode_token(token, settings.JWT_SECRET_KEY, settings.JWT_ALGORITHM)`
5. 验证 `payload["type"] == "access"`
6. 提取 `uid = payload["uid"]`
7. 通过 `AdminUserRepository(db).get_by_uid(uid)` 查询管理员
8. 管理员不存在时抛出 `AuthenticationException(detail="管理员不存在")`
9. 调用 `set_uid(admin.uid)` 设置 observability context
10. 返回 `AdminUser` 实例

### get_optional_current_admin

逻辑：try/except 包装 `get_current_admin`，捕获 `AuthenticationException` 和 `InvalidTokenException` 返回 `None`。

### get_request_meta

逻辑：从 `Request` 提取 `(ip_address, user_agent)` 元组。
- `ip_address = request.client.host if request.client else None`
- `user_agent = request.headers.get("user-agent")`

Import：
- `from api_service.core.config import settings`
- `from api_service.core.db import get_db`
- `from api_service.models import AdminUser`
- `from api_service.repositories.admin_user_repository import AdminUserRepository`
- `from api_service.common.security.jwt import decode_token, get_token_jti`
- `from api_service.common.security.token_blacklist import is_token_blacklisted`
- `from api_service.common.core.exceptions import AuthenticationException, InvalidTokenException`
- `from api_service.common.observability import set_uid`

定义模块级 `security = HTTPBearer(auto_error=False)`。
</action>
