<objective>
Replace the 5 admin-service `gateways/*.py` HTTP proxy modules with same-process Python service calls. Five new admin services compose Phase 4 user services + Phase 3 repositories + (for inference logs only) HMAC HTTP via Plan 05-01's `common/internal.py`. The five controllers under this plan are ports of admin-service source files with every `_gateway.X(...)` call replaced by `XxxService.X(db, ...)` and every `safe_audit_commit(...)` call replaced by inline `await AdminAuditService.record(...) + await db.commit()` (Pitfall 2 + 13).

Purpose: This plan delivers the "proxy elimination" promise of Phase 5 — admin user-management, dashboard, vouchers, route monitor, and service logs all flow through the merged service WITHOUT a single network hop to user-service or router-service (those two services are now folded into api-service).

Output: 5 new controllers (28 endpoints under `/api/v1/admin/{users,dashboard,vouchers,route-monitor,service-logs}/*`); 5 new admin services (~990 lines composed from gateway-shape + direct repo/service calls); 4 new admin schema modules; 5 new test files. The 5 source `gateways/*.py` files are NOT deleted in Phase 5 — deletion is scheduled for Phase 9/10 final cutover when the old services are decommissioned.
</objective>
