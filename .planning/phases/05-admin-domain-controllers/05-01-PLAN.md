<objective>
This plan is the **gating plan** for Phase 5. It (a) hoists shared response envelopes to `api_service/common/schemas.py` and rewrites every Phase 4 import that points at `api_service.schemas.common`, (b) ports the HMAC sender module from admin-service so Plan 05-03 can use it for inference-service log fetches, (c) ports admin auth + super-admin bootstrap + the bare `AdminAuditService` because Plans 05-02 and 05-03 both depend on `await AdminAuditService.record(...)` in every mutating endpoint, and (d) extends `core/policies.py` with `require_active_admin` / `require_super_admin` plus the two new exception classes those guards raise.

Purpose: Without this plan landing first, Plans 05-02 and 05-03 cannot compile (audit service missing; HMAC sender missing; admin guards missing; common schemas duplicated).

Output: 11 new files + 5 modified files; 3 new test files (`test_admin_auth.py`, `test_admin_bootstrap.py`, `test_schemas_hoist.py`) covering ADMIN-01 + ADMIN-12 + D-04 import shape; admin_router mounted under `/api/v1/admin`; super-admin bootstrap wired into the lifespan registry at priority 25.

**Revision note (2026-05-19, per plan-checker Warning 4):** Task 1 has been split into Task 1a (D-04 hoist + Phase 4 import rewrite — small blast-radius foundational change) and Task 1b (HMAC sender port + admin exceptions + admin policies + 5 new settings keys — larger surface change). Task 2 is unchanged. This split keeps each task within the ~10-30% context budget while preserving the Wave 1 gating contract.
</objective>
