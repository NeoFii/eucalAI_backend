<task type="auto">
  <name>Task 2: E2E relay flow tests for 3 protocols (stream + non-stream)</name>
  <files>services/api-service/tests/integration/test_relay_e2e.py</files>
  <action>
Create E2E relay tests per D-05 through D-08. Each test uses `app_client` fixture with `Authorization: Bearer sk-test-integration-key` header.

Note: Since ASGITransport buffers the full response (per research), SSE tests parse `resp.text` as a complete body rather than iterating chunks.

Tests to implement:

1. `test_openai_chat_non_stream` — POST /v1/chat/completions with `{"model": "gpt-4o-mini", "messages": [{"role": "user", "content": "hi"}], "stream": false}`. Assert 200, response has `choices[0].message.content`, `usage.prompt_tokens > 0`, `usage.completion_tokens > 0`.

2. `test_openai_chat_stream` — Same but `stream: true`. Assert 200, content-type contains "text/event-stream". Parse body: split by `\n\n`, each non-empty chunk starts with `data: `. Verify at least one `data: {json}` chunk has `choices[0].delta`. Verify final chunk is `data: [DONE]`. Verify at least one chunk contains `usage` field (stream_options include_usage).

3. `test_anthropic_messages_non_stream` — POST /v1/anthropic/messages with `{"model": "gpt-4o-mini", "messages": [{"role": "user", "content": "hi"}], "max_tokens": 100, "stream": false}`. Assert 200, response has `content[0].text`, `usage.input_tokens > 0`.

4. `test_anthropic_messages_stream` — Same but `stream: true`. Assert 200, text/event-stream. Parse body: lines with `event: ` and `data: `. Verify `event: message_start` appears. Verify `event: content_block_delta` appears with `delta.text`. Verify `event: message_stop` at end.

5. `test_openai_responses_non_stream` — POST /v1/responses with `{"model": "gpt-4o-mini", "input": "hi", "stream": false}`. Assert 200, response has `output` array with at least one item.

6. `test_openai_responses_stream` — Same but `stream: true`. Assert 200, text/event-stream. Parse body for `event: response.` prefixed events. Verify `response.created` and `response.completed` events present.

7. `test_balance_deducted_after_relay` (per D-08) — Record balance before call (from Redis `user:quota:{user_id}`). Make a non-stream chat call. Wait 0.5s for fire-and-forget settlement. Read balance again. Assert balance decreased.

8. `test_call_log_persisted` — Make a non-stream chat call. Wait 0.5s for fire-and-forget write. Query CallLog table via db_session for the user_id. Assert at least one row exists with correct model name.

9. `test_invalid_api_key_returns_401` — POST /v1/chat/completions with `Authorization: Bearer sk-invalid`. Assert 401.

10. `test_insufficient_balance_returns_402` — Set user balance to 0 in Redis. POST /v1/chat/completions. Assert 402 or 400 with insufficient balance error.

All tests use `@pytest.mark.asyncio` (auto mode handles this). Group related tests in a class `TestRelayE2E` for organization.
